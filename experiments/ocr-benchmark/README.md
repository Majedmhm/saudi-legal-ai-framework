## 1. معلومات الـ PDF / PDF Metadata

| الخاصية | القيمة |
|---------|--------|
| الملف | `sources/judicial-decisions/1435/1.pdf` |
| المنتج | Adobe InDesign CS5.5 (7.5) |
| عدد الصفحات | 502 صفحة |
| حجم الملف | ~12.4 MB |
| نوع المحتوى | نص متجه (vector text) مُضمَّن في InDesign — **ليس مسحًا ضوئيًا تقليديًا** |
| pdftotext | فارغ (الترميز غير قابل للقراءة المباشرة) |

**اكتشاف مهم:** هذا الملف مُنشأ بـ InDesign وليس مسحًا ضوئيًا لورق مطبوع.
الخطوط مُضمَّنة كـ Type3/CID encoding غير قابلة لـ pdftotext — لكنها تُنتج صورة عالية الجودة عبر pdftoppm. هذا أفضل من مسح ضوئي حقيقي لورق.

**Important discovery:** This file was created in InDesign, not scanned from paper.
Fonts are embedded as Type3/CID encoding that pdftotext cannot decode — but they produce a high-quality image via pdftoppm. This is better than a true paper scan.

## 2. الأوامر المستخدمة / Commands Used

```bash
# تثبيت الأدوات / Install tools
brew install poppler          # pdftoppm
brew install tesseract tesseract-lang  # OCR + Arabic model

# تحويل صفحات محددة إلى صور 300 DPI
pdftoppm -r 300 -png -f 488 -l 488 1.pdf results/page488
pdftoppm -r 300 -png -f 50  -l 50  1.pdf results/page050
pdftoppm -r 300 -png -f 121 -l 121 1.pdf results/page121

# تشغيل Tesseract OCR بالعربية
tesseract results/page488-488.png results/ocr-page488-raw -l ara --psm 6
tesseract results/page050-050.png results/ocr-page050-raw -l ara --psm 6
tesseract results/page121-121.png results/ocr-page121-raw -l ara --psm 6

# اختبار إضافي — psm 3 (auto layout) على صفحة الجدول
tesseract results/page488-488.png results/ocr-page488-psm3 -l ara --psm 3
```

**قيم psm المستخدمة / PSM values used:**
- `--psm 6` = Assume a single uniform block of text
- `--psm 3` = Fully automatic page segmentation (no OSD)

### صفحة 488 — الفهرس (جدول) / Index Page (Table)

**ما تحتويه الصفحة:** جدول 4 أعمدة — موضوع القضية · رقم القضية · الرقم التسلسلي · رقم الصفحة.

**نتيجة psm 6:** ❌ فشل شبه كامل
```
تسليم مبلغ حال من ثمن أسهم كرد سس | 86 الدريكا
المؤجلة
لقاصر
وجودرضا
```
4 أسطر فقط من جدول يحتوي ~18 صفاً. أرقام القضايا والصفحات مفقودة تماماً.

**نتيجة psm 3 (auto):** ⚠️ جزئي — أفضل بشكل ملحوظ
- موضوعات القضايا (العمود الأول): مقروءة نسبياً
- قسم "بيع / غبن": ظهر
- أرقام القضايا: فوضى كاملة — أحرف عشوائية وأرقام خاطئة
- أرقام الصفحات (١٥٢/١ ، ١٥٦/١...): ظهرت جزئياً لكن بتشويه

**المشكلة الجوهرية:** OCR يقرأ محتوى الخلايا لكن لا يفهم أن هذه خلايا جدول. بنية الأعمدة مفقودة كلياً — النص يُستخرَج كتدفق خطي لا كجدول.

**تقييم / Rating:** ❌ غير قابل للاستخدام للفهرسة

### صفحة 50 — Reasoning منظَّم / Structured Reasoning

**ما تحتويه الصفحة:** أول صفحة حكم — عنوان الموضوع، بيانات المحكمة، ملخص موضوع القضية، نص الحديث النبوي، ملخص الدعوى.

**نتيجة psm 6:** ⚠️ جزئي

النص المُستخرَج (خام بلا تنظيف):
```
جرح ه11                           ← عنوان القسم — خطأ كامل
الرقم التسلسلي: ‎٠‏              ← الرقم مفقود (بدلاً من ٧)
محكمة الدرجة الأولى: المحكمة العامة بالأحساء  ← صحيح ✅
رقم القضية: ‎"901477٠١‏ تاريخها: ‎١5"‏  ← أرقام مشوَّهة ❌
محكمة الاستئناف: محكمة الاستثناف بالمنطقة الشرقيّة  ← "الاستثناف" بدلاً من "الاستئناف" ❌
```

**الحديث النبوي:** `قول النبي صل الله عليه وسلم: (البيعان بالخيار مالم يتفرقا).` — مقروء ✅

**نص الدعوى (الفقرة الطويلة):** مقروء بشكل معقول — ~75% دقة تقديرية.

```
أقام المدعي دعواه ضد المدعى عليههما طالبا إلزامه| بنقل ملكية نصيبهما  ← "إلزامه|" بدلاً من "إلزامهما"
...ووجود غبن في ثمن المبيع غير مقبول لأن البيع تم بطوعهم| ورضاهما  ← "بطوعهم|" بدلاً من "بطوعهما"
```

**المشاكل المكتشفة:**
| المشكلة | مثال |
|---------|-------|
| الخطوط الزخرفية | عناوين مزخرفة تُقرأ كهراء |
| التشكيل | مفقود كلياً |
| علامات RTL/LTR | ‎ و ‏ تظهر في المخرج كرموز |
| الأرقام الهجرية | `١5"` بدلاً من `١٤٣٣` |
| الضمائر المتصلة | "عليههما" → "عليه|" (| بدلاً من ما) |

**تقييم / Rating:** ⚠️ قابل للاستخدام كمسودة — يحتاج مراجعة بشرية مكثَّفة

### صفحة 121 — نص كثيف بأرقام / Dense Text with Numbers

**ما تحتويه الصفحة:** نص متواصل لمحضر جلسة محكمة — تواريخ، مبالغ مالية، بيانات أطراف مُخفاة بـ (...)، تفاصيل بيع سيارة.

**نتيجة psm 6:** ⚠️ جزئي — أفضل الثلاثة في استخراج الفقرات

**نماذج من المخرج:**
```
المقيدة بالمحكمة برقم ‎7376١١915‏ وتاريخ ‎577/١١/١5‏  ← أرقام مشوَّهة
سعودي الجنسية بموجب السجل المدني رقم (...)          ← صحيح ✅
قام موكلٍ ببيع سيارة (...) غمارتين موديل ‎5٠١7‏       ← "غمارتين" بدلاً من "غماريتن" أو علامة تجارية
مائة وعشرة الآف ريال                                ← "الآف" بدلاً من "الآلاف" ❌
ألف وحخمسائة ريال                                   ← "وحخمسائة" ← حرف إضافي خاطئ
ستون ألفا وخمساثة ريال                              ← "وخمساثة" بدلاً من "وخمسمائة" ❌
```

**أرقام الأقساط والتواريخ:** مُستخرَجة جزئياً لكن لا يمكن الاعتماد عليها دون مراجعة.

**المشاكل المكتشفة:**
| المشكلة | مثال |
|---------|-------|
| الأرقام المدمجة في نص | علامات RTL تُشوِّه السياق |
| كلمات المبالغ | "خمسمائة" → "خمساثة"، "خمسائة" |
| التواريخ الهجرية | "١5797/07/7١هه" (هه بدلاً من ه) |
| بيانات المحكمة | أرقام القضايا مقروءة جزئياً |

**تقييم / Rating:** ⚠️ قابل للاستخدام كمسودة — الأرقام والمبالغ تحتاج تحقق خاص

## 5. ملخص اكتشافات الجودة / Quality Summary

| المعيار | الدرجة | التفصيل |

| **التواريخ الهجرية** | ⚠️ جزئي | القرن الأول صحيح أحياناً، التفاصيل مشوَّهة |

## 7. هل أرقام المواد صحيحة؟ / Are Article Numbers Correct?

**ملاحظة مهمة:** صفحات هذا الجزء (المجلد 1 — مدني) تستند بشكل رئيسي إلى الفقه والحديث، لا إلى مواد نظامية صريحة بالترقيم (م/X).

في النماذج المُختبَرة: لم تظهر أرقام مواد نظامية (مادة رقم X من نظام Y). ظهرت أرقام قضايا وتواريخ — وهي مشوَّهة في معظمها.

**للأقسام التي تحتوي مواد نظامية صريحة** (مثل الإعسار والعقار): يجب اختبار صفحاتها تحديداً — النتيجة قد تختلف.

## 9. هل نكمل بهذه الأداة أو لا؟ / Go / No-Go Decision

### القرار: ⚠️ مشروط — Conditional Proceed

```
✅ استمرار — Tesseract مقبول لـ:
  · استخراج مسودة أولى من نصوص الأحكام المتواصلة
  · تحديد بداية ونهاية الأحكام (الرقم التسلسلي، اسم المحكمة)
  · تسهيل القراءة البشرية (قارئ يُصحح بدلاً من يكتب من الصفر)

❌ لا يكفي وحده لـ:
  · استخراج أرقام القضايا والتواريخ بثقة
  · استخراج بنية الجداول (الفهرس)
  · الاعتماد على أي رقم في السياق القانوني
  · أي محتوى يعتمد على التشكيل

الشرط: كل مخرج يبقى ocr_draft / unverified حتى تتم مراجعته بشريًا مقابل الصورة.
```

### ما يجب فعله قبل الاستخدام الفعلي / Before Production Use

```
1. اختبار صفحة تحتوي مواد نظامية صريحة (م/X) للتحقق من دقة الأرقام النظامية
2. اختبار PaddleOCR كبديل — خاصة على صفحات الجداول
3. تطوير خط معالجة مسبقة (preprocessing): deskew + contrast + binarization
4. تحديد معيار الدقة المقبول قبل الترقية من ocr_draft إلى ocr_reviewed
```
